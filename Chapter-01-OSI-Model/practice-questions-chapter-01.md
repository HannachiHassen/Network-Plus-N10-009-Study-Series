## Single Answer — Q1 through Q30

---

**Q1. Answer: D — 7**
> The OSI (Open Systems Interconnection) model has exactly 7 layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application.

---

**Q2. Answer: C — Network**
> The Network layer (Layer 3) handles logical (IP) addressing and routing. It determines the best path for data to travel between different networks. Routers operate at this layer.

---

**Q3. Answer: D — Physical**
> The Physical layer (Layer 1) converts data into electrical signals, light pulses, or radio waves for transmission over a physical medium. It deals with bits, cables, voltages, and hardware specifications.

---

**Q4. Answer: C — Transport**
> The Transport layer (Layer 4) provides end-to-end communication between hosts, including error recovery, flow control, and segmentation. TCP and UDP operate here.

---

**Q5. Answer: C — Presentation**
> The Presentation layer (Layer 6) handles data translation, formatting, encryption, and compression. Garbled characters suggest a character encoding or format conversion issue — a classic Presentation layer problem.

---

**Q6. Answer: D — Frame**
> PDUs by layer: Application/Presentation/Session = Data | Transport = Segment | Network = Packet | Data Link = **Frame** | Physical = Bit. Memorize these.

---

**Q7. Answer: B — Session**
> The Session layer (Layer 5) establishes, manages, and terminates communication sessions between applications. It handles session checkpointing and recovery.

---

**Q8. Answer: C — Switch**
> Switches operate at Layer 2 (Data Link) and use MAC addresses to forward frames within a local network. Routers = Layer 3. Hubs = Layer 1.

---

**Q9. Answer: C — DNS**
> DNS (Domain Name System) is an Application layer (Layer 7) protocol. TCP and UDP are Layer 4. IP is Layer 3. ARP is Layer 2/3 boundary.

---

**Q10. Answer: C — Port number header (segment)**
> At the Transport layer, a header containing source and destination **port numbers** is added, creating a **segment**. The IP header (with IP addresses) is added at Layer 3. The MAC header is added at Layer 2.

---

**Q11. Answer: B — TCP/IP Network Interface = OSI Physical + Data Link**
> The TCP/IP Network Interface layer (also called Link layer) maps to OSI Layers 1 (Physical) and 2 (Data Link). The TCP/IP Internet layer maps to OSI Layer 3 (Network) only.

---

**Q12. Answer: B — 4**
> The TCP/IP model has 4 layers: Application, Transport, Internet, and Network Interface.

---

**Q13. Answer: D — Application**
> The TCP/IP Application layer consolidates three OSI layers: Session (5), Presentation (6), and Application (7) into a single layer.

---

**Q14. Answer: C — Layer 3**
> Routers operate primarily at Layer 3 (Network) of the OSI model. They use IP addresses to make forwarding decisions and route packets between different networks.

---

**Q15. Answer: B — Application, Presentation, Session, Transport, Network, Data Link, Physical**
> From top (Layer 7) to bottom (Layer 1): **A**ll **P**eople **S**eem **T**o **N**eed **D**ata **P**rocessing.

---

**Q16. Answer: D — ARP**
> ARP (Address Resolution Protocol) maps known IP addresses to MAC addresses within the same network. It operates at the boundary of Layers 2 and 3. DNS maps domain names to IPs. ICMP sends error messages.

---

**Q17. Answer: C — ICMP**
> The `ping` command uses **ICMP** (Internet Control Message Protocol) echo request and echo reply messages to test connectivity. ICMP operates at Layer 3.

---

**Q18. Answer: D — Hub**
> Hubs are Layer 1 (Physical) devices. They simply repeat electrical signals to all connected ports with no intelligence. Switches = Layer 2. Routers = Layer 3. Bridges = Layer 2.

---

**Q19. Answer: B — Physical → Data Link → Network → Transport → Application**
> De-encapsulation works bottom-up on the receiving host. The Physical layer receives bits → Data Link removes the frame header → Network removes the IP header → Transport removes the port header → Application processes the data.

---

**Q20. Answer: D — Presentation**
> The Presentation layer handles data formatting, translation, encryption/decryption, and compression/decompression. It ensures data from the Application layer is in a usable format.

---

**Q21. Answer: C — Layer 3**
> Routers make forwarding decisions based on IP addresses, which is a Layer 3 (Network) function. Each router reads the destination IP in the packet header to determine the next hop.

---

**Q22. Answer: D — Packet**
> Layer 3 (Network) PDU = **Packet**. Layer 4 = Segment. Layer 2 = Frame. Layer 1 = Bit.

---

**Q23. Answer: D — Application**
> HTTPS is an Application layer (Layer 7) protocol. Although it uses TLS (Presentation layer) for encryption, the protocol itself — HTTP over TLS — is classified at Layer 7.

---

**Q24. Answer: B — Data Link**
> MAC addresses are a Layer 2 (Data Link) concept. Switches use MAC address tables to forward frames between devices on the same network segment.

---

**Q25. Answer: C — Internet**
> The TCP/IP Internet layer handles IP addressing and routing — equivalent to OSI Layer 3 (Network). It is responsible for packet routing across different networks.

---

**Q26. Answer: C — Presentation**
> SSL/TLS primarily operates at Layer 6 (Presentation) of the OSI model. It handles encryption and decryption of data before it is passed to the Application layer. Note: Some sources also cite Layer 4 or Layer 5 — on the exam, **Presentation** is the expected answer.

---

**Q27. Answer: B — Layer 2**
> Port security is a feature of managed switches. It operates at Layer 2 (Data Link) by restricting which MAC addresses can communicate through a specific switch port.

---

**Q28. Answer: C — Provides node-to-node data transfer and handles MAC addressing within the same network**
> The Data Link layer handles node-to-node communication within a single network segment using MAC addresses. It also provides error detection via frame check sequences (FCS/CRC).

---

**Q29. Answer: D — Network Interface**
> The TCP/IP Network Interface layer (also called the Link layer) is responsible for the physical transmission of data. It corresponds to OSI Layers 1 (Physical) and 2 (Data Link).

---

**Q30. Answer: B — SMTP**
> SMTP (Simple Mail Transfer Protocol) is a Layer 7 Application layer protocol used for sending email. IP = Layer 3. Ethernet = Layer 2. TCP = Layer 4.

---

## Multiple Answer — Q31 through Q40

---

**Q31. Answer: B — TCP, C — UDP**
> TCP and UDP are the two primary Transport layer (Layer 4) protocols. IP is Layer 3. ICMP is Layer 3. HTTP is Layer 7.

---

**Q32. Answer: B — Switch, D — Bridge**
> Switches and bridges both operate at Layer 2 (Data Link) using MAC addresses. Hubs and repeaters = Layer 1. Routers = Layer 3.

---

**Q33. Answer: B — Segmentation and reassembly, C — Flow control, D — Error detection and correction**
> The Transport layer segments data, controls flow to prevent congestion, and detects/corrects transmission errors. Logical/IP addressing = Layer 3. Encryption = Layer 6. MAC addressing = Layer 2.

---

**Q34. Answer: B — The OSI model has 7 layers; the TCP/IP model has 4, C — TCP/IP combines OSI's Application, Presentation, and Session layers**
> The OSI model = 7 layers. TCP/IP model = 4 layers. TCP/IP collapses OSI's top 3 layers (Application, Presentation, Session) into one Application layer. TCP/IP was developed from ARPANET; OSI was developed by ISO.

---

**Q35. Answer: C — IP, D — ICMP**
> IP (Internet Protocol) and ICMP (Internet Control Message Protocol) both operate at Layer 3 (Network). Ethernet = Layer 2. TCP = Layer 4. FTP = Layer 7.

---

**Q36. Answer: A, B, D**
> A: Each layer adds headers (and sometimes trailers). B: The Data Link layer uniquely adds both a header AND a trailer (FCS/CRC for error checking). D: The Physical layer converts data into raw bits. C is wrong — encapsulation happens on the **sending** host. E is wrong — the Transport layer adds port numbers, not MAC addresses.

---

**Q37. Answer: A — HTTP, C — DNS, E — FTP**
> HTTP, DNS, and FTP are all Application layer protocols in both OSI and TCP/IP models. TCP = Transport layer. IP = Internet layer in TCP/IP.

---

**Q38. Answer: B — ARP resolves IP addresses to MAC addresses, D — ARP operates at the Network/Data Link layer boundary**
> ARP resolves known IP addresses to MAC addresses and operates at the Layer 2/3 boundary. It uses broadcasts, not TCP. DNS resolves domain names to IPs — that is NOT ARP's job.

---

**Q39. Answer: B — Multilayer Switch, D — Firewall**
> Multilayer switches operate at Layer 2 AND Layer 3 (and sometimes higher). Firewalls can operate from Layer 3 up to Layer 7 (NGFWs). Hubs and repeaters = Layer 1 only. Bridges = Layer 2 only.

---

**Q40. Answer: A — Defines electrical/optical/mechanical characteristics, C — Hubs and repeaters operate at this layer**
> The Physical layer defines how bits are transmitted over physical media. Hubs and repeaters are Layer 1 devices. MAC addressing = Layer 2. IP addresses = Layer 3. End-to-end error recovery = Layer 4.

## Scenario-Based — Q41 through Q47

---

**Q41. Answer: C — Layer 7 — DNS resolution failure**
> The user can ping by IP (Layer 3 is fine) but cannot resolve domain names. This is a **DNS** issue — a Layer 7 Application layer service. DNS translates domain names to IP addresses; without it, name resolution fails.

---

**Q42. Answer: C — Layer 3**
> IP addresses are Layer 3 (Network layer) addressing. Duplicate IP addresses cause conflicts at the Network layer. MAC address conflicts = Layer 2. Port conflicts = Layer 4.

---

**Q43. Answer: B — Data Link layer — verify MAC addressing and switch configuration**
> In a bottom-up troubleshooting approach, after confirming Layer 1 (Physical) is functioning, the next step is Layer 2 (Data Link). This involves checking switch port status, MAC address tables, duplex settings, and VLAN configuration.

---

**Q44. Answer: D — Next-Generation Firewall (NGFW)**
> A standard router inspects up to Layer 3. A stateful firewall inspects up to Layer 4. A **Next-Generation Firewall (NGFW)** performs deep packet inspection up to Layer 7, inspecting application payloads to detect and block malicious content.

---

**Q45. Answer: C — Broadcast — Layer 2**
> A destination MAC address of `FF:FF:FF:FF:FF:FF` is a Layer 2 **broadcast** — sent to all devices on the local network segment. ARP requests use this broadcast to find the MAC address for a given IP.

---

**Q46. Answer: C — Layer 6 — Presentation**
> TLS (Transport Layer Security), which HTTPS uses for encryption, operates at the **Presentation layer** (Layer 6) of the OSI model. The Presentation layer handles encryption, decryption, and data formatting.

---

**Q47. Answer: C — Each layer wraps the data from the layer above with its own header, creating a new PDU**
> This is the definition of **encapsulation**. As data travels down the OSI model, each layer adds its own header (and sometimes trailer) to the data from the layer above it, creating a new PDU at each stage.

---

## Fill in the Blank — Q48 through Q50

---

**Q48. Answer: Data Link**
> The **Data Link** layer (Layer 2) uses MAC (Media Access Control) addresses for physical/hardware addressing within a local network segment.

---

**Q49. Answer: Application**
> In the TCP/IP model, the **Application** layer encompasses the functions of OSI Layers 5 (Session), 6 (Presentation), and 7 (Application).

---

**Q50. Answer: Encapsulation / De-encapsulation**
> **Encapsulation** is the process of adding headers (and sometimes trailers) at each layer on the sending host as data moves down the OSI stack. **De-encapsulation** is the reverse — removing those headers layer by layer on the receiving host as data moves up the stack.
